Color Scheme

AX 인터페이스는 사용자, AI, 시스템 세 주체의 발화와 상태를 시각적으로 구분해야 합니다. Color Scheme 은 의미 기반 토큰으로 이 구분을 만들고, 라이트/다크 모드에서 동일하게 작동하도록 설계되어 어떤 컨텍스트에서도 안정적인 정보 위계를 보장합니다.

Anatomy

Label·Surface·Line·Fill·Status 다섯 카테고리로 분리되어 컴포넌트가 의미를 명확히 표현합니다. 모든 토큰은 라이트와 다크 두 모드에서 서로 매핑된 페어로 정의되어 컴포넌트는 별도 분기 없이 작동합니다.

AI 응답

시맨틱 토큰으로 어떤 컴포넌트도 두 모드를 자연스럽게 지원합니다.

OS 설정 또는 사용자의 명시 선택에 따라 토큰의 실제 값이 자동 전환됩니다. 라이트 → 다크로 전환되더라도 컴포넌트는 토큰 이름만 참조하므로 동일한 의미·위계가 유지됩니다.

AI 응답 상태(생성 중·완료·오류)는 색으로 즉시 구분합니다. 사용자는 화면 변화만 보고도 시스템이 무엇을 하는지 알 수 있어야 합니다. 색은 항상 보조 시그널(아이콘·텍스트)과 함께 사용해 색맹·저시력 사용자도 정보를 잃지 않도록 합니다.

How to use

label-strong

시맨틱 토큰 사용

컴포넌트가 의미 토큰만 참조하면 모드 전환이 자동으로 작동합니다.

Do

시맨틱 토큰(label-strong, surface-elevated)을 사용하여 의미와 모드 적응을 모두 보장합니다.

#171719 ← hardcoded

Primitive 직접 사용

다크 모드에선 검은 글씨가 그대로 남아 가독성이 깨집니다.

Don't

Primitive 컬러(#171719)를 컴포넌트에서 직접 사용하지 않습니다. 다크 모드 적응이 깨집니다.
